# Plano: Backend do Módulo de Agenda (Appointments)

## Contexto
O modelo `Appointment` já existe no schema Prisma com campos: `id`, `enterpriseId`, `date`, `status`, `notes`, `patientId`, `dentistId`. Precisamos criar a camada de serviços, controllers e rotas para CRUD completo, com foco em filtragem por data (dia/mês/ano).

## Arquivos a Criar

### 1. `backend/src/services/Appointments/index.ts`
- **`CreateAppointmentParams`**: `{ date, patientId, dentistId, notes?, status? }`
- **`createAppointment(params, enterpriseId)`**: Valida se patientId e dentistId existem na mesma enterprise; cria agendamento
- **`listAppointments(enterpriseId, filters)`**: Lista agendamentos com filtros opcionais: `date` (dia específico), `year`, `month`. Inclui dados de `patient` (name, phone) e `dentist` (name)
- **`getAppointmentById(id, enterpriseId)`**: Busca agendamento por ID com relações
- **`updateAppointment(id, params, enterpriseId)`**: Atualiza campos permitidos (date, status, notes, patientId, dentistId)
- **`deleteAppointment(id, enterpriseId)`**: Remove agendamento

### 2. `backend/src/controllers/Appointments/index.ts`
- Funções: `create`, `list`, `getById`, `update`, `remove`
- Validação de campos obrigatórios (patientId, dentistId, date)
- Validação de `date` como string ISO válida
- Validação de `status` como enum válido
- Acesso a `req.user!.enterpriseId` para multi-tenancy
- Tratamento de erros seguindo padrão existente

### 3. `backend/src/routes/Appointments/index.ts`
```
POST   /              → create (authMiddleware)
GET    /              → list (authMiddleware)
GET    /:id           → getById (authMiddleware)
PUT    /:id           → update (authMiddleware)
DELETE /:id           → remove (authMiddleware)
```

### 4. `backend/src/index.ts` (edição)
- Importar e registrar: `app.use("/api/appointments", appointmentsRouter)`

## Endpoints da API

### POST /api/appointments
Body: `{ date: "2025-01-15T10:00:00.000Z", patientId, dentistId, notes?, status? }`
Response: `{ id, date, status, notes, patient: {...}, dentist: {...}, ... }`

### GET /api/appointments?date=2025-01-15
Query params opcionais: `date` (YYYY-MM-DD), `year` (YYYY), `month` (MM)
Response: `[{ id, date, status, patient: { name, phone }, dentist: { name }, ... }]`

### GET /api/appointments/:id
Response: `{ id, date, status, notes, patient: {...}, dentist: {...}, createdAt, ... }`

### PUT /api/appointments/:id
Body: `{ date?, status?, notes?, patientId?, dentistId? }`
Response: agendamento atualizado

### DELETE /api/appointments/:id
Response: `{ message: "Agendamento removido com sucesso." }`
